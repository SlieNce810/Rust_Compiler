# 编译器设计文档

## 项目背景

本编译器的目标是设计一个能够从源代码生成目标平台可执行代码的编译器。编译器将支持语言的词法分析、语法分析、语义分析、中间代码生成、优化、目标代码生成及链接等功能。该编译器适用于开发特定目标平台（例如 STM32F403、ESP32 等）的嵌入式程序。

---

## 1. 编译器概述

### 1.1 编译器概述

编译器将负责以下任务：

* 从源代码中解析出语法结构，并生成语法树。
* 执行语法检查和类型检查。
* 将高层语言转化为平台无关的中间代码。
* 对中间代码进行优化，减少冗余操作。
* 将优化后的中间代码生成目标平台的汇编代码。
* 链接目标文件，生成可执行程序。

### 1.2 编译器结构

编译器包括以下模块：

* **前端**：负责词法分析、语法分析、语义分析。
* **中端**：负责中间代码生成和优化。
* **后端**：负责目标代码生成、汇编代码生成。
* **链接器**：将多个目标文件链接成最终的可执行文件或固件。

### 1.3 支持的平台

* **STM32F403**（ARM Cortex-M4架构）
* **ESP32**（Tensilica Xtensa架构）

## 2. 语言设计

### 2.1 语法设计

我们将设计一种简单的面向嵌入式的语言，支持常见的控制流、数据类型和函数定义。示例语法规则如下：

* **数据类型**：`int`、`float`、`bool`。
* **控制结构**：`if`、`while`、`for`、`return`。
* **函数**：支持带参函数定义和调用。
* **表达式**：支持算术运算符（`+`、`-`、`*`、`/`）、关系运算符（`<`、`>`、`==`）。

### 2.2 语法规则（BNF表示法）

```
<program> ::= <function>*

<function> ::= "func" <type> <id> "(" <params> ")" <block>

<params> ::= <param> | <param> "," <params>
<param> ::= <type> <id>

<block> ::= "{" <statement>* "}"
<statement> ::= <declaration> | <assignment> | <if_statement> | <while_statement> | <return_statement>

<declaration> ::= <type> <id> ";"
<assignment> ::= <id> "=" <expression> ";"
<if_statement> ::= "if" "(" <expression> ")" <block> [ "else" <block> ]
<while_statement> ::= "while" "(" <expression> ")" <block>
<return_statement> ::= "return" <expression> ";"

<expression> ::= <term> | <expression> "+" <term> | <expression> "-" <term>
<term> ::= <factor> | <term> "*" <factor> | <term> "/" <factor>
<factor> ::= <id> | <number> | "(" <expression> ")"

<id> ::= [a-zA-Z_][a-zA-Z0-9_]*
<number> ::= [0-9]+
<type> ::= "int" | "float" | "bool"
```

### 2.3 语义规则

* **类型检查**：确保变量在使用前声明，且类型匹配。
* **作用域分析**：确保函数内部的变量不会与外部变量冲突。
* **函数调用检查**：确保函数调用时传递的参数类型与声明一致。

## 3. 编译器模块

### 3.1 词法分析器（Lexer）

#### 3.1.1 功能

词法分析器的主要任务是将源代码的字符流分解成标记（tokens）。每个标记代表一种语言的元素，如关键字、标识符、运算符等。

#### 3.1.2 输入与输出

* **输入**：源代码的字符流（例如从文件中读取）。
* **输出**：标记（Token）的列表，每个标记包含类型和相应的值。

#### 3.1.3 设计

* 使用正则表达式来匹配语言的各种元素。
* 将字符流转换为标记列表。

**工具**：使用 Rust 的 `regex` 库来实现。

#### 3.1.4 代码实现

```rust
use regex::Regex;

#[derive(Debug)]
enum Token {
    Keyword(String),
    Identifier(String),
    Operator(String),
    Number(i32),
    // 其他标记类型
}

fn lexer(input: &str) -> Vec<Token> {
    let mut tokens = Vec::new();
    let re = Regex::new(r"\b(int|float|bool|if|while|return)\b|\b[a-zA-Z_][a-zA-Z0-9_]*\b|\d+|[+\-*/=;(){}]").unwrap();
    for cap in re.captures_iter(input) {
        if let Some(m) = cap.get(0) {
            let token_str = m.as_str();
            if token_str == "int" || token_str == "float" || token_str == "bool" {
                tokens.push(Token::Keyword(token_str.to_string()));
            } else if token_str.chars().all(char::is_alphanumeric) {
                tokens.push(Token::Identifier(token_str.to_string()));
            } else if token_str == "+" || token_str == "-" || token_str == "*" || token_str == "/" {
                tokens.push(Token::Operator(token_str.to_string()));
            } else if token_str.parse::<i32>().is_ok() {
                tokens.push(Token::Number(token_str.parse().unwrap()));
            }
        }
    }
    tokens
}
```

### 3.2 语法分析器（Parser）

#### 3.2.1 功能

语法分析器的任务是将词法分析器输出的标记流转化为抽象语法树（AST），并检查语法是否符合定义的语法规则。

#### 3.2.2 输入与输出

* **输入**：标记（Token）列表。
* **输出**：抽象语法树（AST）。

#### 3.2.3 设计

* 使用递归下降解析法来处理语法规则。
* 每个语法规则对应一个函数，递归调用实现规则匹配。

### 3.3 语义分析器（Semantic Analyzer）

#### 3.3.1 功能

语义分析的主要任务是检查程序中的类型、作用域、函数调用等语义错误。

#### 3.3.2 输入与输出

* **输入**：抽象语法树（AST）。
* **输出**：符号表、类型检查结果。

#### 3.3.3 设计

* 维护符号表，记录变量和函数的类型信息。
* 执行类型检查、作用域分析、函数调用验证等。

### 3.4 中间代码生成器（IR Generator）

#### 3.4.1 功能

生成平台无关的中间代码，常用的中间代码形式有 **三地址码（TAC）** 和 **静态单赋值（SSA）**。

#### 3.4.2 输入与输出

* **输入**：抽象语法树（AST）。
* **输出**：中间代码（TAC 或 SSA）。

#### 3.4.3 设计

* 将 AST 转换为三地址码，每个操作表示为 `t1 = a + b` 的形式。
* 支持表达式、赋值、条件语句等。
